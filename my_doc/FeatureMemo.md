## 1. 自律思考 & 自発発話

**ステータス: 実装済み**

### 実装済み内容

- `MoodManager`（`firmware/src/mood/MoodManager.h/cpp`）による気分パラメータ管理
- `wantToTalk` が閾値（1.0）を超えたら `spontaneousSpeech()` で自発発話
- 会話発生時に `wantToTalk` リセット・`sleepiness` を 0.3 減算
- 自発発話時に `wantToTalk` リセット（sleepiness はリセットしない）
- LLM レスポンスの `[META]{"emotion":"happy","joy":0.3,"trust":0.1}[/META]` を解析し joy/trust を更新
- HeadTouch スワイプで `joy +0.1` / `trust +0.05`
- 撫でた時に表情を一時変更（2秒間）: Sad/Angry → Neutral、それ以外 → Happy
- アイドル時に `getDominantExpression()` の表情を Avatar に適用

### 気分パラメータ

| パラメータ | 範囲 | 更新ルール |
|---|---|---|
| joy（楽しい⇔悲しい） | -1.0〜1.0 | 会話の [META] タグで変動・時間経過で減衰（±1.0→0.0 約10分）・スワイプで +0.1 |
| trust（信頼⇔怒り） | -1.0〜1.0 | 会話の [META] タグで変動・時間減衰なし（信頼・怒りは持続）・スワイプで +0.05 |
| sleepiness（眠気） | 0.0〜1.0 | 時間経過で増加（0→0.8 約10分）・会話で -0.3 |
| wantToTalk（話したい度） | 0.0〜1.0 | 時間経過で増加（0→1.0 約5分）・発話/会話でゼロリセット |

**更新周期**: 30秒（deltaSeconds で補正するため増加量は周期に依存しない）

### 表情マッピング

最も強いパラメータが表情を決定：

| 条件 | 表情 | エフェクト |
|---|---|---|
| sleepiness >= 1.0 | Sleeping（就寝） | Zzz（右上→左下・大中小） |
| sleepiness が最大かつ >= 0.8 | Sleepy | 泡マーク |
| joy が最大かつ >= 0.6 | Happy | ハート |
| joy が最大かつ <= -0.6 | Sad | 寒さマーク |
| trust が最大かつ >= 0.6 | Happy | ハート |
| trust が最大かつ <= -0.6 | Angry | 怒りマーク |
| それ以外 | Neutral | なし |

### 就寝・起床

| 項目 | 内容 |
|---|---|
| 就寝トリガー | sleepiness >= 1.0 |
| 就寝中の動作 | Sleeping 表情・頭を正面に固定・きょろきょろ停止・自発発話/音声認識を無効化 |
| 起床トリガー | HeadTouch（LongPress / SwipeForward / SwipeBackward）いずれか |
| 起床時の処理 | sleepiness・wantToTalk をゼロリセット・負の joy/trust を半分回復（悲しみ・怒りが覚める）・きょろきょろ再開 |

### 自発発話プロンプト

会話履歴がある場合：
```
あなたはかわいいロボットのスタックちゃんです。
あなた自身の気分は「〇〇」です。
過去の会話の話題を踏まえつつ、自然に話しかけてください。
```

会話履歴がない場合（起動直後など）：
```
あなたはかわいいロボットのスタックちゃんです。
あなた自身の気分は「〇〇」です。
この前提で自由に話しかけてください。
```

### LLM への感情指示（ChatGPT.cpp）

毎回のプロンプトに以下を付加し、LLM に joy/trust の変化量を返させる：

```
返答の末尾に感情と気分変化を付与してください。
形式: [META]{"emotion": "happy", "joy": 0.3, "trust": 0.1}[/META]
joy: 楽しさの変化量（-1.0〜+1.0）
trust: 信頼感の変化量（-1.0〜+1.0）
```

### 残タスク

- [ ] **sleepiness と wantToTalk の競合問題**: wantToTalk が先に閾値に達して自発発話すると会話扱いにならず sleepiness がリセットされないため、眠い状態のまま自発発話し続ける可能性がある
- [ ] **話題ストック**: 会話から話題キーワードを抽出してリングバッファに保持し、自発発話の話題として使う

### 設計メモ

- 気分更新はローカル計算のみ。LLM は自発発話時だけ呼ぶ（API コスト削減）
- `isBusy()` フラグで会話中への割り込みを防止
- 機能5（アイドル演出）と連動予定：機嫌が低い→Sleepy表情多め、話したい度が高い→落ち着きなく目線が動く

## 2. ウェイクワード後の音源定位・顔追従

**ステータス: 未実装**

### 概要

ウェイクワードを検知したら音の方向を推定してそちらを向き、カメラで顔を見つけたら見つめ続ける。

### 詳細

- **音源定位**: ウェイクワード検知後、マイク入力から音の到来方向を推定してサーボで頭を向ける
- **顔検出**: カメラで正面付近を探索し、顔が見つかった方向にサーボを調整する
- **顔追従**: 顔が画角内にある間は顔の位置を追い続けて見つめる
- **会話終了後**: 顔が画角から外れたら正面に戻る

### 実装イメージ

```
[ウェイクワード検知]
  ↓
音源方向を推定 → サーボで頭をその方向へ向ける
  ↓
カメラで顔検出を開始
  ↓
顔が見つかった場合: 顔の位置にサーボを微調整して見つめる
顔が見つからない場合: 音源方向のまま保持
  ↓
通常の会話フローへ（会話中も顔追従を継続）
  ↓
会話終了・顔ロストで正面に戻る
```

### 実装メモ

- CoreS3 はデュアルマイク（ES7210）搭載。TDOA で「左・右」の粗い推定で十分
- 顔検出は `esp32-camera` ライブラリの内蔵モデルで追加ハードウェア不要。1〜2fps に落とすと負荷を抑えられる
- 顔追従サーボ制御は既存の `servo` 処理に乗せられる
- 既存の `ENABLE_CAMERA` フラグ・`Camera.h` との兼ね合いを確認

## 4. 就寝・起床と夢の発話

**ステータス: 部分実装済み（就寝・起床音声・あくびは実装済み。夢の発話は未実装）**

### 実装済み内容

- **あくび**: sleepiness >= 0.8 で1サイクルに1回 `yawn_fuwa.mp3` を再生
- **就寝音声**: sleepiness >= 1.0 で1サイクルに1回ランダム再生（`sleep_oyasumi` / `sleep_oyasuminasai`）
- **起床音声**: 起床時にランダム再生（`wakeup_ohayou` / `wakeup_ohayougozaimasu`）
- 音声ファイルは SD カード `/ack/{speakerId}/` に配置（相槌と同じ仕組み）
- VoiceVox 生成スクリプト（`generate_ack_sounds.ps1`）に yawn / sleep / wakeup を追加済み

### 残タスク

- [ ] **夢の発話**: 就寝時に会話ログを LLM に渡して夢のストーリーを生成し、起床時に発話する

### 実装メモ

- 夢の生成は就寝時に行う（起床直前でもよいが就寝タイミングが自然）
- 「話しかけたら起きる」実装はスリープ中も STT または簡易ウェイクワード検出が必要

## 5. アイドル時の生体感演出

**ステータス: 部分実装済み（きょろきょろは実装済み。その他は未実装）**

### 実装済み

- **きょろきょろ（IdleLookAround）**: ランダムな方向にサーボで頭を振り向く
  - 水平 ±70度・垂直 ±30度
  - キープ時間 1〜7秒
  - 会話中は停止・会話終了後に再開

- **facialLoop（Avatar ライブラリ）**: 自動サッカード・まばたき・呼吸アニメーション（常時動作）

### 残タスク（未実装）

アイドル中はいくつかの動作をランダムに組み合わせて実行する：

| 動作 | 内容 | 頻度イメージ |
|---|---|---|
| 表情変化 | 気分パラメータに応じた表情をふらっと切り替える | ときどき |
| 顔発見 → 微笑む | カメラで顔を検出したら Happy 表情で数秒見つめ、その後見回しに戻る | 顔検出時 |

### 実装メモ

- `AiStackChanMod.cpp` の `idle()` 内に追加するのが自然な実装場所
- 各動作に重みを持たせた抽選にすると「目線が多め、頭の動きは少なめ」などチューニングしやすい
- 機能1の気分パラメータと連動：機嫌が低い→動作が鈍い、話したい度が高い→落ち着きなく動く

## 6. ベクトル検索による長期記憶

**ステータス: 未実装（機能1の会話ログ基盤と組み合わせると効果的）**

### 概要

会話の記憶を意味的に検索できる長期記憶システム。既存の `enableMemory`（要約テキストを1つ保持するだけ）より大規模かつ精度の高い記憶を実現する。

### 設計方針

**ベクトル生成は OpenAI Embeddings API に委託し、検索計算のみ ESP32 で行う。**

```
[会話終了後]
  会話テキスト → Embeddings API → ベクトル → SD カードに追記保存

[会話開始時]
  発話テキスト → Embeddings API → クエリベクトル
  → SD カードのベクトルと比較（ESP32 で計算）
  → 類似度上位 N 件をプロンプトに注入 → ChatGPT へ送信
```

### 実装メモ

- `text-embedding-3-small` が軽量・低コストでおすすめ
- 256 次元に削減すると SD・RAM を節約できる
- Embeddings API 呼び出しは非同期化で会話後の遅延を隠せる

## 7. 頭部 LED による感情可視化

**ステータス: 未実装**

### 概要

頭の左右についている LED を使い、MoodManager の joy（楽しい⇔悲しい）と trust（信頼⇔怒り）の2軸パラメータをリアルタイムに表示する。

### 設計案（検討中）

| LED | 軸 | 表示方法（例） |
|---|---|---|
| 左 LED | joy（楽しい⇔悲しい） | 正値 → 暖色（黄/橙）、負値 → 寒色（青）、輝度で強度 |
| 右 LED | trust（信頼⇔怒り） | 正値 → 緑、負値 → 赤、輝度で強度 |

### 実装メモ

- LED の制御方法（GPIO・I2C・IO 拡張経由など）を先に確認する必要あり
- PY32L020 IO 拡張（0x6F）経由の可能性もある
- 感情更新（30秒周期）に合わせて LED を更新すればよい
